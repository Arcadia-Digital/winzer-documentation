# Contributing

## Branching
- Create feature branches from `main`: `feature/<short-description>`
- Keep changes scoped and reviewable

## Pull Requests
- Describe the change, rationale, and user impact
- Link to relevant investigation or documentation sections
- Ensure no secrets are added; run basic lint locally if applicable

## Coding Style
- Follow existing patterns in each stack
- Prefer small, focused edits; avoid mass formatting changes

## Documentation
- Update `*.md` and `*.html` where behavior changes
- Add examples and troubleshooting where useful

# Cursor Quick Start for Winzer Development

**TL;DR**: Get productive with Cursor in 10 minutes

---

## ⚡ Quick Setup (5 minutes)

1. **Install Cursor**: Download from [cursor.sh](https://cursor.sh)
2. **Clone Repo**: `git clone https://github.com/petebuzzell-ad/winzer-documentation.git`
3. **Open in Cursor**: File → Open Folder
4. **Add Rules**: Settings → Cursor → Rules → Paste contents of `.cursorrules`
5. **Done!** You're ready to code.

---

## 🎯 Essential Commands

| Shortcut       | Action          | Use For                        |
| -------------- | --------------- | ------------------------------ |
| `Cmd/Ctrl + K` | Inline edit     | Quick code changes             |
| `Cmd/Ctrl + L` | Chat            | Questions, debugging           |
| `Cmd/Ctrl + I` | Composer        | Multi-file edits               |
| `@filename`    | Reference file  | Mention specific files in chat |
| `#search`      | Codebase search | Find code patterns             |

---

## 🚀 Your First 3 Tasks

### 1. Understand the Project (Cmd/Ctrl + L)
```
What is the Winzer platform? Give me a high-level overview.
```

### 2. Explore the Codebase (Cmd/Ctrl + L)
```
Walk me through what happens when a CSV is uploaded to the middleware.
```

### 3. Make a Change (Cmd/Ctrl + K)
Select any comment in the code and ask:
```
Rewrite this comment to be more descriptive
```

## 💡 Common Workflows

### Adding a Feature
1. Use `#` to search for similar features
2. Use `Cmd/Ctrl + L` to understand context
3. Use `Cmd/Ctrl + K` to implement with AI help
4. Test and iterate

### Fixing a Bug
1. Open relevant files
2. Use `Cmd/Ctrl + L`: "Why is this failing? Show me the issue"
3. Use `Cmd/Ctrl + K` to apply the fix
4. Test with investigation files

### Writing Docs
1. Use `Cmd/Ctrl + L`: "Help me write documentation for [feature]"
2. Use AI to draft
3. Refine with `Cmd/Ctrl + K`
4. Review for clarity

---

## 🎓 Pro Tips

- **Be specific**: "Add weight validation" not "Fix this"
- **Use context**: `@file.js` to reference specific files
- **Iterate**: Start broad, narrow down with follow-ups
- **Review AI code**: Don't blindly accept suggestions
- **Learn patterns**: Notice how AI solves similar problems
